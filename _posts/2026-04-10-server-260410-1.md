## 1. Cron이란

Cron은 리눅스에서 특정 시간마다 작업을 자동 실행하는 스케줄러다.

예시:

```
*/5 * * * * php script.php
```

의미:

- 5분마다 `script.php` 실행

---

## 2. Cron의 문제점

Cron은 이전 작업이 끝났는지 확인하지 않는다.

즉,

- 작업이 오래 걸리면
- 다음 실행이 겹치면서 중복 실행이 발생한다

이로 인해:

- DB 중복 처리
- 데이터 정합성 문제
- 예상치 못한 장애

가 발생할 수 있다.

---

## 3. flock으로 중복 실행 방지

### flock이란

flock은 file lock의 약자로, 파일을 기준으로 잠금을 걸어 중복 실행을 막는다.

### 사용 예시

```
flock -n /tmp/cron_job.lock php script.php
```

동작 방식:

1. `/tmp/cron_job.lock` 파일에 lock 시도
2. 이미 실행 중이면 실행하지 않고 종료
3. 실행 중이 아니면 script 실행
4. 작업 종료 시 lock 자동 해제

---

### 핵심 개념

- PHP 파일을 잠그는 것이 아니다
- 별도의 `.lock` 파일을 기준으로 실행을 제어한다
- lock은 프로세스 종료 시 자동으로 해제된다

---

## 4. 쉘 블록으로 여러 명령 묶기

Cron에서 여러 명령을 함께 실행하려면 `{}`를 사용한다.

```
{ 명령어1; 명령어2; 명령어3; }
```

각 명령은 `;`로 구분한다.

---

## 5. 로그 기록하기

```
>> /path/to/logfile.log 2>&1
```

설명:

- `>>` : 로그 파일에 추가 기록
- `2>&1` : 에러 메시지도 함께 기록

---

## 6. 날짜별 로그 생성

```
cron_exec_$(date +\%Y-\%m-\%d).log
```

예:

```
cron_exec_2026-04-10.log
```

---

## 7. 오래된 로그 삭제

```
find /path -name "cron_exec_*.log" -mtime +30 -delete
```

설명:

- 30일 지난 로그 파일 삭제

---

## 8. 최종 예제

```
*/5 * * * * { flock -n /tmp/cron_auto_penalty.lock /usr/bin/php74 /var/www/html/ksep_cron/cron_auto_penalty.php >> /var/www/html/ksep_cron/cron_logs/cron_exec_$(date +\%Y-\%m-\%d).log 2>&1; find /var/www/html/ksep_cron/cron_logs -name "cron_exec_*.log" -mtime +30 -delete; }
```

---

## 9. 전체 동작 흐름

1. flock으로 lock 시도
2. 성공하면 PHP 실행
3. 실행 결과 로그 기록
4. 오래된 로그 삭제
5. 작업 종료 시 lock 해제

---

## 10. 실무 팁

- lock 파일은 `/tmp`에 두는 것이 일반적이다
- `n` 옵션을 사용하면 중복 실행 시 바로 종료된다
- 로그는 날짜별로 나누는 것이 관리하기 쉽다

---

## 정리

Cron은 단순히 작업을 반복 실행할 뿐이다.

중복 실행 방지는 flock으로 해결해야 하며,

로그와 정리 정책까지 포함해야 안정적인 운영이 가능하다.
