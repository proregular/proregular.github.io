## 1️⃣ 기본 개념

- 일반적으로 PHP는 `.php` 확장자를 가진 파일만 **PHP 엔진** 또는 **FPM 풀**로 전달되어 처리됨
- `.html`, `.htm`, `.js` 같은 정적 파일은 **웹 서버가 직접 반환**
- PHP로 처리하려면 **웹 서버 설정** 또는 **FPM/Apache 핸들러**를 추가해야 함

---

## 2️⃣ Apache에서 PHP 외 확장자 처리

### 2-1. mod_php 사용 시

- `AddHandler` 또는 `AddType`을 이용해 PHP로 처리할 확장자를 지정

```
# .php 뿐만 아니라 .html도 PHP로 처리
AddHandler application/x-httpd-php .php .html
```

- 단점:
    - 모든 `.html` 요청이 PHP 엔진을 거침 → CPU 부담 증가
    - 보안 취약점 가능성 ↑
    - 보통은 **정적 파일은 그대로 두고, 필요한 동적 파일만 PHP 처리** 권장

---

### 2-2. PHP-FPM(FastCGI) 사용 시

- VirtualHost 안에서 `SetHandler`로 확장자별 PHP-FPM 연결 가능

```
<FilesMatch "\.(php|html)$">
    SetHandler "proxy:unix:/run/php74-fpm.sock|fcgi://localhost/"
</FilesMatch>
```

- 설명:
    - `FilesMatch` → 지정한 확장자를 PHP-FPM으로 전달
    - `.php` + `.html` 요청 모두 FPM 풀에서 실행
- 주의:
    - 보안 때문에 **필요한 경우에만 확장자 추가**
    - 정적 파일은 굳이 PHP 처리하지 않는 것이 안전

---

## 3️⃣ PHP-FPM 내부 제한 (`security.limit_extensions`)

- `www.conf` 파일에 있는 설정으로 **허용할 확장자 제한 가능**

```
; www.conf 예시
security.limit_extensions = .php
```

- 의미:
    - PHP-FPM 풀에서 **이 확장자만 PHP로 처리** 가능
    - `.html` 등 추가하려면 여기에도 확장자를 넣어야 함

```
security.limit_extensions = .php .html
```

- CLI PHP나 `php.ini`에는 해당 설정이 없음 → **FPM 풀에서만 적용**

---

## 4️⃣ php.ini에서는 안 됨

- `php.ini`는 PHP 동작 관련 설정만 담당
- **어떤 확장자를 PHP로 처리할지는 웹 서버/ FPM 풀에서 설정**
- [www.conf](http://www.conf/) 또는 Apache/Nginx 핸들러 설정에서 처리

---

## 5️⃣ 정리: 확장자 처리 정책

| 방법 | 설정 위치 | 특징 | 권장 여부 |
| --- | --- | --- | --- |
| mod_php | Apache `AddHandler` / `AddType` | 모든 요청이 PHP 엔진 거침 | 작은 서버에서만, 주로 .php만 |
| PHP-FPM | Apache `SetHandler` + FPM `security.limit_extensions` | 지정한 확장자만 PHP-FPM으로 전달 | 필요할 때만 .html 추가 가능 |
| php.ini | 없음 | PHP 설정만 담당 | 확장자 지정 X |
- **권장 패턴:**
    - `.php` → PHP 처리
    - `.html` → 정적 반환 (보안/성능 이유)
    - 필요하면 `.html` 등만 제한적으로 FPM 풀에 추가

---

💡 **포인트 요약**

1. PHP로 처리할 확장자는 **웹 서버 / FPM 설정**에서만 지정 가능
2. `security.limit_extensions` → FPM 풀 단위 확장자 제한
3. php.ini → 확장자 지정 불가
4. 정적 파일은 가능한 **웹 서버에서 직접 반환** → 성능 및 보안 유리
