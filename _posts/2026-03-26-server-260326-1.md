웹 애플리케이션에서 파일 업로드 기능은 자주 사용되지만, 동시에 보안 취약점이 가장 많이 발생하는 영역 중 하나이다.

검증이 미흡할 경우 웹쉘 업로드 등 심각한 공격으로 이어질 수 있다.

본 문서는 파일 업로드 취약점을 점검하기 위한 코드 분석 방법과 검색 전략을 정리한 것이다.

---

## 1. 전체 코드 검색 (Find in Files)

파일 업로드 취약점 점검의 첫 단계는 프로젝트 전체에서 관련 코드 위치를 식별하는 것이다.

### 1.1 에디트플러스 사용

- 메뉴: Search → Find in Files
- Directory: 프로젝트 루트
- File types:
    
    *.php;*.inc;*.html
    

---

### 1.2 리눅스 환경 (권장)

리눅스에서는 grep을 활용하면 빠르게 검색할 수 있다.

```
grep -rn "move_uploaded_file" ./
grep -rn "$_FILES" ./
grep -rn "file_put_contents" ./
grep -rn "copy(" ./
```

---

## 2. 핵심 탐색 대상

### 2.1 업로드 직접 관련 (최우선)

```
$_FILES
move_uploaded_file
```

- $_FILES: 업로드 데이터 접근 지점
- move_uploaded_file: 실제 파일 저장 지점

---

### 2.2 파일 생성 및 저장 함수

```

```

---

### 2.3 업로드 경로 및 디렉토리

```
upload
uploads
path
dir
mkdir
```

업로드 경로가 웹에서 직접 접근 가능한지 반드시 확인해야 한다.

---

### 2.4 파일명 및 확장자 처리

```

```

확장자 및 MIME 타입 검증 여부를 확인한다.

---

### 2.6 위험한 코드 패턴

```

```

확인 사항:

- 원본 파일명을 그대로 사용하는지 여부
- MIME 타입을 그대로 신뢰하는지 여부

---

```

```

위 확장자가 허용되는 경우 웹쉘 업로드 가능성이 매우 높다.

## 3. 취약한 코드 예시

```
move_uploaded_file($_FILES['file']['tmp_name'], "upload/" . $_FILES['file']['name']);
```

문제점:

- 확장자 검증이 없음
- 원본 파일명을 그대로 사용
- 웹 접근 가능한 경로에 저장

---

## 4. 안전한 구현 예시

```
$ext = strtolower(pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION));
$allow = ['jpg', 'png', 'pdf'];

if (in_array($ext, $allow)) {
    $newName = uniqid() . "." . $ext;
    move_uploaded_file($_FILES['file']['tmp_name'], "upload/" . $newName);
}
```

보안 요소:

- 확장자 화이트리스트 적용
- 파일명 랜덤화
- 원본 파일명 미사용

---

## 5. 점검 체크리스트

파일 업로드 코드 발견 시 다음 항목을 반드시 확인해야 한다.

- 확장자 제한 존재 여부
- 파일명 랜덤 처리 여부
- 업로드 경로의 웹 접근 가능 여부
- MIME 타입 검증 여부
- 실행 파일 업로드 가능 여부

---

## 결론

파일 업로드 취약점 점검의 핵심은 다음 세 가지이다.

- 파일이 생성되는 모든 코드 추적
- 업로드 입력값($_FILES) 처리 방식 분석
- 확장자 및 파일명 검증 로직 확인

이 세 가지를 중심으로 점검하면 대부분의 취약점을 효과적으로 식별할 수 있다.
