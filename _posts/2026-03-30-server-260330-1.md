리눅스에서 파일과 디렉토리의 권한, 사용자 및 그룹 관리에 대한 기본 개념을 정리한다. 이 내용을 이해하면 웹 서버 환경에서 PHP로 파일 업로드, 임시 디렉토리 관리, 공유 폴더 설정 등을 안전하게 설계할 수 있다.

---

## 1. 사용자(User)와 그룹(Group)

### 사용자

- 리눅스에서 각 프로세스는 **특정 사용자 계정**으로 실행된다.
- Apache(httpd) 웹서버 설치 시 대부분 배포판에서는 자동으로 전용 사용자와 그룹이 생성된다.

| 배포판 | 기본 사용자 | 기본 그룹 |
| --- | --- | --- |
| CentOS / RHEL | apache | apache |
| Ubuntu / Debian | www-data | www-data |
- PHP에서 `mkdir`이나 파일 업로드를 수행하면, 생성되는 파일의 소유자(user)와 그룹(group)은 **웹서버 프로세스의 계정**을 따른다.

### 그룹

- 그룹은 여러 사용자를 묶어 파일/디렉토리 권한을 공유하기 위한 단위이다.
- 한 사용자는 **하나의 기본 그룹(primary group)**과 **여러 보조 그룹(supplementary groups)**에 속할 수 있다.
- 예를 들어, Apache 계정을 다른 공유 그룹에 추가하면 그 그룹에 속한 사용자도 해당 파일/디렉토리에 접근할 수 있다.

```bash
id apache
# 출력 예시
uid=48(apache) gid=48(apache) groups=48(apache),100(webgroup)
```

- gid=48 → 기본 그룹
- groups=48(apache),100(webgroup) → 보조 그룹 포함

## 2. 파일과 디렉토리 권한

리눅스에서 파일 및 디렉토리 권한은 `rwx`로 표시되며, 타입에 따라 의미가 다르다.

### 일반 파일

| 권한 | 의미 |
| --- | --- |
| r | 파일 내용 읽기 가능 |
| w | 파일 내용 쓰기 가능 |
| x | 프로그램/스크립트 실행 가능 (사진, 텍스트 파일은 일반적으로 필요 없음) |

### 디렉토리

| 권한 | 의미 |
| --- | --- |
| r | 디렉토리 안 파일 목록 조회 가능 (`ls`) |
| w | 파일 생성, 삭제 가능 |
| x | 디렉토리 접근 가능 (`cd`) |
- 디렉토리는 읽기만으로는 접근할 수 없고, **실제 접근하려면 실행(x) 권한 필요**
- 사진(jpg, png)이나 텍스트(txt) 파일은 `x` 권한이 없어도 열거나 읽을 수 있다.

---

## 3. 권한 확인 명령어

### 기본 명령어

- `ls -l 파일명` : 파일/디렉토리 권한, 소유자, 그룹 확인
- `stat 파일명` : 권한과 소유자 정보 포함 상세 정보 확인
- `namei -l 경로` : 경로 전체에 걸친 디렉토리 권한 확인

예시:

```bash
ls -l /UPDIR/zip/tmp_test
-rw-r--r-- 1 apache apache 1024 Mar 30 12:00 image.jpg
drwxr-xr-x 2 apache apache 4096 Mar 30 12:00 tmp_test
```

- 파일 권한: `rw-r--r--`
- 디렉토리 권한: `drwxr-xr-x`

---

## 4. 그룹 공유와 SetGID

- 파일은 하나의 그룹만 소유할 수 있다.
- 여러 사용자가 파일에 접근하도록 하려면:
    1. **보조 그룹에 사용자 추가**
    2. **디렉토리 SetGID 설정**: 새 파일이 부모 디렉토리의 그룹을 상속받음

```bash
# SetGID 비트 설정
sudo chmod -R 2775 /UPDIR/zip
# 디렉토리 소유자와 그룹 변경
sudo chown -R apache:webgroup /UPDIR/zip
```

- 이렇게 하면 Apache가 생성한 파일도 **webgroup** 소속이 되며, 그룹 사용자들이 읽기/쓰기 가능하게 된다.

---

## 5. 권한과 파일 유형 정리

| 파일/디렉토리 | r | w | x | 비고 |
| --- | --- | --- | --- | --- |
| 일반 파일(txt, jpg) | 읽기 가능 | 쓰기 가능 | 실행 필요 없음 | 스크립트/프로그램 제외 |
| 디렉토리 | 목록 확인 | 생성/삭제 가능 | cd 가능 | 실행(x) 권한 필요 |

---

## 6. 정리

1. Apache 설치 시 자동 생성된 사용자/그룹이 PHP 파일 업로드의 기본 소유자/그룹이 된다.
2. 한 사용자는 **기본 그룹 + 여러 보조 그룹**에 속할 수 있다.
3. 파일은 **하나의 그룹만 소유 가능**, 여러 그룹 접근은 보조 그룹 활용 또는 ACL 필요.
4. 일반 파일은 **r(읽기), w(쓰기)만 필요**하며, x(실행)는 사진/텍스트 파일에는 의미 없음.
5. 디렉토리 공유를 위해 SetGID와 그룹 권한을 활용하면 여러 사용자가 안전하게 파일을 공유할 수 있다.
